Sentiment Analysis on News Using Transformers.js in Vue

Run Hugging Face transformer models directly in the browser — no Python backend, no API keys. Build a Vue.js app that classifies news headlines as positive, negative, or neutral in real time.

1. Why Run ML in the Browser?

Traditional ML inference requires a backend server — you send text to an API, a Python service runs the model, and you get results back. This works, but it adds latency, server costs, and complexity.

What if the model ran entirely in the user's browser?

  • Zero backend costs — no GPU servers, no API billing
  • Privacy-first — user data never leaves their device
  • Offline capable — works after the model is cached
  • Instant deployment — ship as a static site on GitHub Pages

This is exactly what Transformers.js enables. It brings Hugging Face's transformer models to JavaScript, running inference via ONNX Runtime in the browser.

2. What is Transformers.js?

Transformers.js is a JavaScript library by Hugging Face that lets you run pre-trained transformer models directly in the browser (or Node.js). Under the hood, it uses ONNX Runtime Web — meaning models are compiled to the ONNX format and executed via WebAssembly or WebGPU.

💡
Key insight: Transformers.js provides a pipeline API almost identical to Python's transformers library. If you've used pipeline("sentiment-analysis") in Python, the JavaScript version is nearly the same.

For sentiment analysis, we'll use the sentiment-analysis pipeline with the Xenova/distilbert-base-uncased-finetuned-sst-2-english model — a lightweight, quantized model optimized for browser use (~67MB).

3. Architecture Overview

Here's how the pieces fit together:

Data flow: Vue UI → Web Worker → Transformers.js → ONNX Runtime → Results

The key design decision is running the model in a Web Worker. Transformer inference is CPU-intensive — doing it on the main thread would freeze the UI. The Web Worker runs in a separate thread, keeping the Vue app responsive while the model processes headlines.

During the Vue CLI prompts, select:

5. Setting Up the Web Worker

The Web Worker is the bridge between your Vue UI and Transformers.js. It loads the model once, caches it, and processes incoming text.

src/workers/sentiment.worker.js
import { pipeline } from '@huggingface/transformers';

// Cache the pipeline instance
let classifier = null;

// Initialize the sentiment analysis pipeline
async function loadModel() {
  if (!classifier) {
    self.postMessage({
      type: 'status',
      message: 'Loading model...'
    });

    classifier = await pipeline(
      'sentiment-analysis',
      'Xenova/distilbert-base-uncased-finetuned-sst-2-english',
      { progress_callback: (progress) => {
          self.postMessage({
            type: 'loading-progress',
            progress
          });
        }
      }
    );

    self.postMessage({
      type: 'status',
      message: 'Model ready!'
    });
  }
  return classifier;
}

// Listen for messages from the main thread
self.addEventListener('message', async (event) => {
  const { type, texts, id } = event.data;

  if (type === 'analyze') {
    try {
      const model = await loadModel();
      const results = [];

      for (let i = 0; i < texts.length; i++) {
        const result = await model(texts[i]);
        results.push({
          text: texts[i],
          label: result[0].label,
          score: result[0].score
        });

        // Report progress for batch processing
        self.postMessage({
          type: 'progress',
          current: i + 1,
          total: texts.length,
          id
        });
      }

      self.postMessage({
        type: 'result',
        results,
        id
      });
    } catch (error) {
      self.postMessage({
        type: 'error',
        message: error.message,
        id
      });
    }
  }
});

Key design choices here:

  • Model caching — classifier is loaded once and reused. The ONNX model files are also cached in the browser's Cache API by Transformers.js, so subsequent page loads are near-instant.
  • Progress reporting — We send loading progress and per-item batch progress back to the UI, enabling a responsive loading bar.
  • Error handling — Errors in the worker are caught and forwarded to the main thread instead of silently failing.

6. Creating the Vue Composable

The composable wraps the Worker communication in a clean reactive API — useSentiment(). Any Vue component can call it without worrying about message passing or worker lifecycle.

src/composables/useSentiment.js
import { ref, shallowRef } from 'vue';
import SentimentWorker from '../workers/sentiment.worker.js?worker';

export function useSentiment() {
  const results = shallowRef([]);
  const isLoading = ref(false);
  const isModelReady = ref(false);
  const progress = ref({ current: 0, total: 0 });
  const status = ref('Initializing...');
  const error = ref(null);
  const loadingProgress = ref(0);

  // Create the worker
  const worker = new SentimentWorker();
  let requestId = 0;

  // Listen for worker messages
  worker.addEventListener('message', (event) => {
    const data = event.data;

    switch (data.type) {
      case 'status':
        status.value = data.message;
        if (data.message === 'Model ready!') {
          isModelReady.value = true;
        }
        break;
      case 'loading-progress':
        if (data.progress?.progress) {
          loadingProgress.value = Math.round(data.progress.progress);
        }
        break;
      case 'progress':
        progress.value = {
          current: data.current,
          total: data.total
        };
        break;
      case 'result':
        results.value = data.results;
        isLoading.value = false;
        status.value = `Analyzed ${data.results.length} headlines`;
        break;
      case 'error':
        error.value = data.message;
        isLoading.value = false;
        break;
    }
  });

  // Public API
  function analyze(texts) {
    if (!texts.length) return;
    isLoading.value = true;
    error.value = null;
    progress.value = { current: 0, total: texts.length };
    const id = ++requestId;
    worker.postMessage({ type: 'analyze', texts, id });
  }

  return {
    results,
    isLoading,
    isModelReady,
    progress,
    status,
    error,
    loadingProgress,
    analyze
  };
}
📦
Why shallowRef for results? We use shallowRef instead of ref for the results array because Vue doesn't need to deeply observe each result object — we replace the entire array on each analysis. This avoids unnecessary reactivity overhead with large batches.

7. Building the UI Components

The app has three main areas: a headline input (textarea + sample headlines), a results display with color-coded sentiment cards, and a summary dashboard.

The Main App Component

src/App.vue
<script setup>
import { ref, computed } from 'vue'
import { useSentiment } from './composables/useSentiment'

const {
  results, isLoading, isModelReady,
  progress, status, error, loadingProgress, analyze
} = useSentiment()

const inputText = ref('')

// Sample headlines for instant demo
const sampleHeadlines = [
  'Tech stocks surge as AI adoption accelerates across industries',
  'Devastating earthquake hits coastal region, thousands displaced',
  'Scientists announce breakthrough in renewable energy storage',
  'Global supply chain disruptions threaten holiday shopping season',
  'Local community raises $2M for children\'s hospital expansion',
  'Cybersecurity breach exposes millions of user accounts',
  'New study reveals promising results for Alzheimer\'s treatment',
  'Market volatility continues amid rising interest rate concerns'
]

function loadSamples() {
  inputText.value = sampleHeadlines.join('\n')
}

function handleAnalyze() {
  const lines = inputText.value
    .split('\n')
    .map(l => l.trim())
    .filter(l => l.length > 0)
  analyze(lines)
}

// Computed summary stats
const summary = computed(() => {
  if (!results.value.length) return null
  const pos = results.value.filter(r => r.label === 'POSITIVE').length
  const neg = results.value.length - pos
  return { positive: pos, negative: neg, total: results.value.length }
})
</script>

Sentiment Result Cards

Each headline gets a color-coded card showing the sentiment label and confidence score:

Template (in App.vue)
<div class="results-grid">
  <div
    v-for="(item, i) in results"
    :key="i"
    class="result-card"
    :class="item.label === 'POSITIVE' ? 'result-card--pos' : 'result-card--neg'"
  >
    <div class="result-card__header">
      <span class="result-card__label">
        {{ item.label === 'POSITIVE' ? '😊' : '😟' }}
        {{ item.label }}
      </span>
      <span class="result-card__score">
        {{ (item.score * 100).toFixed(1) }}%
      </span>
    </div>
    <p class="result-card__text">{{ item.text }}</p>
    <!-- Confidence bar -->
    <div class="result-card__bar">
      <div
        class="result-card__bar-fill"
        :style="{ width: (item.score * 100) + '%' }"
      />
    </div>
  </div>
</div>

The confidence bar is a simple CSS-driven visualization — the width is bound to the score percentage. Cards with POSITIVE sentiment get a green accent; NEGATIVE gets red. This makes scanning results intuitive at a glance.

8. Batch Analysis with Progress

When analyzing multiple headlines, the worker processes them one at a time and reports progress. We surface this as a progress bar:

Progress bar template
<div v-if="isLoading" class="progress-section">
  <div class="progress-bar">
    <div
      class="progress-bar__fill"
      :style="{ width: (progress.current / progress.total * 100) + '%' }"
    />
  </div>
  <p class="progress-text">
    Analyzing {{ progress.current }} / {{ progress.total }} headlines...
  </p>
</div>
⚠️
First load is slow. The model (~67MB) needs to download on first use. Transformers.js caches it in the browser's Cache API, so subsequent loads are much faster (typically under 2 seconds). Show a loading indicator with download progress to set user expectations.

9. Performance Tips

Here are practical optimizations I've found useful when working with Transformers.js:

  1. Use quantized models. The Xenova/ prefix on Hugging Face denotes models converted and quantized for browser use. They're 4–8x smaller than full models with minimal quality loss.
  2. Preload the model. Trigger loadModel() on app mount rather than waiting for the first analysis. This hides the loading time during the period where the user is reading the UI.
  3. Batch wisely. Processing 100+ headlines sequentially in a Web Worker works fine, but update the UI between items. Don't block the worker's message loop.
  4. Enable WebGPU when available. Transformers.js v3+ supports WebGPU acceleration. On supported browsers (Chrome 113+), this can be 5–10x faster than WASM:
WebGPU configuration
classifier = await pipeline(
  'sentiment-analysis',
  'Xenova/distilbert-base-uncased-finetuned-sst-2-english',
  {
    device: 'webgpu',     // Use GPU if available
    dtype: 'fp32',        // Full precision for WebGPU
  }
);
  1. Cache results. If the same headline is analyzed twice, return the cached result instead of running inference again. A simple Map keyed by text works well.

10. Conclusion & Next Steps

We built a fully client-side sentiment analysis app using Vue.js and Transformers.js. The entire ML inference happens in the browser — no backend, no API keys, no server costs. The key ingredients:

  • Transformers.js for the ML pipeline (Hugging Face in the browser)
  • Web Worker to keep the UI responsive during inference
  • Vue composable for clean, reactive state management
  • ONNX Runtime (WASM/WebGPU) for efficient model execution

Ideas for extending this:

  • Multi-language support — swap in a multilingual model like Xenova/bert-base-multilingual-uncased-sentiment
  • Named Entity Recognition — add NER to extract people, companies, and locations from headlines
  • Trend visualization — chart sentiment over time using a news RSS feed
  • Emotion classification — go beyond positive/negative with fine-grained emotions (joy, anger, fear, etc.)
  • Comparison mode — compare sentiment across different news sources on the same topic
The full source code is available on GitHub. Clone it, experiment with different models, and deploy your own version on GitHub Pages — it's free to host since there's no backend.
